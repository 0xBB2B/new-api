# 额度重置用户可见性

## 目的

重置会收回未用完的余额；提前告知用户"何时重置、重置为多少"，避免余额变动被误认为系统故障或额度丢失。

## 逻辑

1. 用户有生效重置规则时，个人页展示两项信息：
   - **下次重置时间**：按生效规则的周期计算出的下一个触发时点（服务器本地时区）。
   - **重置值**：生效规则的重置值，按站点当前的额度展示方式（货币或 token 数）呈现。
2. 用户无生效规则（全局关闭且无专属规则、带退出自动重置标记、被禁用）时，个人页不出现任何重置相关信息。
3. 展示的信息来自该用户自己的生效规则：专属规则用户看到专属规则的周期与值，而非全局规则的。

## 约束

- 展示的下次重置时间必须与实际定时触发时点一致：daily 为明日 00:00 或今日 00:00 之后最近一次，weekly 为下周一 00:00，monthly 为下月 1 日 00:00（均为服务器本地时区）。
- 展示的重置值与该用户实际被重置到的数值一致。
- 无生效规则的用户页面不渲染重置信息区块，而非渲染空值。
- 展示文案走前端 i18n，随站点语言切换。

## 例子

今天是 2026-07-06（周一）。bob 的专属规则：weekly，重置值 100000；站点额度展示方式为 USD（500000 quota = $1）。

- bob 个人页显示：下次重置时间 2026-07-13 00:00，重置为 $0.20。
- carol（带退出自动重置标记）的个人页：无任何重置信息。
- 管理员将 bob 的专属规则删除且全局规则关闭 → bob 个人页的重置信息消失。

## 验收

- [ ] 适用全局规则的用户看到全局规则的下次触发时点与重置值；专属规则用户看到专属规则的。
- [ ] 展示的下次重置时间与定时触发的实际执行时点一致。
- [ ] 退出自动重置标记、禁用、无任何规则三种情况下个人页均不出现重置信息。
- [ ] 重置值按站点额度展示方式换算呈现，展示文案支持多语言。
