# 额度重置动作语义

## 目的

定义对单个用户执行一次额度重置时，哪些数据改变、哪些数据保持不变、留下什么审计痕迹。

## 逻辑

一次重置动作以"用户 + 生效规则的重置值"为输入，执行：

1. 将用户的 quota **覆盖写**为重置值——不是累加、不是补差，与重置前余额多少无关。
2. used_quota、request_count 等累计统计字段保持不变。
3. 写一条该用户的系统日志，内容包含：重置前余额、重置后余额、触发方式（定时 / 手动）。

重置值是以 quota 内部计费单位表示的整数，允许为 0（该用户当期无预算），不允许负数。

## 约束

- 重置后用户 quota 恒等于重置值，与重置前余额（更高、更低、为负）无关。
- 重置动作不修改 used_quota、request_count、aff_quota 及其他任何用户字段。
- 重置值 < 0 的规则拒绝保存；重置值为 0 合法。
- 每执行一次重置动作恰好产生一条日志；被跳过的用户（无生效规则）不产生日志。
- 重置动作幂等：对同一用户以同一重置值连续执行多次，最终余额与执行一次相同。

## 例子

用户 alice：quota = 123456，used_quota = 876544。生效规则重置值 500000，定时触发执行重置：

- quota：123456 → 500000（未用完的 123456 被收回，不结转）。
- used_quota：876544（不变）。
- 新增一条 alice 的系统日志：重置前 123456，重置后 500000，触发方式为定时。

同一规则下用户 frank：quota = 800000（高于重置值）→ 重置后同样为 500000，多出部分被收回。

## 验收

- [ ] 重置后 quota 精确等于重置值，重置前余额低于、高于重置值两种情况结果一致。
- [ ] 重置前后 used_quota 与 request_count 数值不变。
- [ ] 重置值为 0 的规则可保存且执行后用户 quota 为 0；重置值为负数的规则保存被拒绝。
- [ ] 每次重置产生恰好一条日志，含重置前余额、重置后余额、触发方式。
- [ ] 对同一用户重复执行同一重置，余额与单次执行一致。
