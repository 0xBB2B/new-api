# 额度重置定时触发

## 目的

定义额度重置规则的周期取值与触发时点计算方式，使"什么时候重置"唯一可判定。

## 逻辑

每条额度重置规则（全局或每用户专属）的周期字段取值为三选一：

| 周期 | 触发时点 |
|---|---|
| daily | 每天 00:00 |
| weekly | 每周一 00:00 |
| monthly | 每月 1 日 00:00 |

触发时点按**服务器本地时区**计算。到达触发时点时，系统对每个生效规则周期与之匹配的用户执行一次重置。

服务在触发时点未运行（宕机、重启跨过时点）时，该次触发不执行、恢复后不自动补执行；需要补救时由管理员使用手动触发入口。

## 约束

- 周期字段只接受 daily / weekly / monthly 三个值，其他值拒绝保存。
- 同一触发时点对同一用户至多执行一次重置；重复调度不得造成同一时点的第二次日志记录。
- 触发时点的判定不依赖 UTC：服务器本地时区为 Asia/Shanghai 时，daily 规则在本地 00:00（UTC 前一日 16:00）触发。
- 恢复运行后不回溯任何已错过的触发时点，包括跨多个周期的停机。

## 例子

服务器本地时区 Asia/Shanghai。用户 bob 的专属规则：weekly，重置值 100000。

- 周一 00:00 服务正常运行 → bob 的 quota 被重置为 100000。
- 下一个周一 00:00 前 10 分钟服务停机，周一 01:00 恢复 → 该次触发不执行，bob 的 quota 保持停机前的值；再下一个周一 00:00 正常触发。

## 验收

- [ ] daily / weekly / monthly 规则分别在本地每天 00:00、周一 00:00、每月 1 日 00:00 触发重置。
- [ ] 周期字段传入三选一之外的值时保存被拒绝并报错。
- [ ] 同一触发时点对同一用户只产生一次重置与一条日志。
- [ ] 服务跨过触发时点后恢复，不发生补执行，用户余额不变。
