# 额度重置规则解析

## 目的

确定任意时刻对任意用户"生效的额度重置规则是哪条"，使定时与手动重置的作用对象唯一可判定。

## 逻辑

系统存在两级额度重置规则：

1. **全局规则**：站点级配置，含启用开关、周期、重置值，默认关闭。
2. **每用户专属规则**：按用户配置，含周期、重置值，仅管理员可设置。

对单个用户解析生效规则，按以下顺序判定，命中即停：

1. 用户带退出自动重置标记 → 无生效规则。
2. 用户状态非启用（禁用、已注销）→ 无生效规则。
3. 用户配置了专属规则 → 专属规则生效（周期与重置值都用专属规则的，不与全局字段混合）。
4. 全局规则已启用 → 全局规则生效。
5. 以上都不满足 → 无生效规则。

无生效规则的用户不参与任何自动或手动重置，个人页也不展示重置信息。

## 约束

- 专属规则对全局规则是整体覆盖：用户有专属规则时，全局规则的任何字段（周期、重置值）都不参与该用户的重置。
- 退出自动重置标记优先于专属规则：同时存在时该用户无生效规则。
- 管理员与普通用户按同一解析顺序判定，角色不影响结果。
- 用户从禁用恢复为启用后，重新按上述顺序解析，下一个触发时点起参与重置。
- 全局规则关闭且用户无专属规则时，系统对该用户零行为（不重置、不记日志、不展示）。

## 例子

站点全局规则：启用，每月，重置值 500000。用户情况与解析结果：

| 用户 | 专属规则 | 退出自动重置标记 | 状态 | 生效规则 |
|---|---|---|---|---|
| alice | 无 | 无 | 启用 | 全局（每月，500000） |
| bob | 每周，100000 | 无 | 启用 | 专属（每周，100000） |
| carol | 每周，100000 | 有 | 启用 | 无 |
| dave | 无 | 无 | 禁用 | 无 |

若随后全局规则被关闭：alice 变为无生效规则；bob 不受影响，仍按专属规则重置。

## 验收

- [ ] 全局规则启用时，无专属规则、无退出标记的启用用户按全局规则重置。
- [ ] 有专属规则的用户按专属规则的周期与重置值重置，全局字段不掺入。
- [ ] 带退出自动重置标记的用户在定时与手动重置中都被跳过，即使其配有专属规则。
- [ ] 禁用用户不被重置；恢复启用后从下一个触发时点起重新参与。
- [ ] 全局规则关闭后，仅有专属规则的用户继续被重置，其余用户零行为。
