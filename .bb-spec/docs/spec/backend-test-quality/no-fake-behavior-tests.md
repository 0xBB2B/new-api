# 禁伪行为测试

## 目的

确保每条测试都在保护一个可失败的具体契约，而不是「代码能跑到最后就算通过」。

## 逻辑

伪测试的共同特征：输入是不可复现的随机值、断言只检查「有值/长度大于零/没 panic」、或靠 `time.Sleep` 等一段时间就宣告成功。它们通过率与代码正确性脱钩，坏的实现也会绿。

约束把这些手段整体从 `Test*` 中排除：

- 随机输入放去 `Benchmark*` 做性能压测。
- 等待异步事件用 channel + `time.After` 超时兜底，不用固定 sleep。
- 循环规模控制在能表达契约所需的最小值。
- 表达「可能失败但不能崩」的场景用真实错误注入（假 server 返回 4xx、故意错的 JSON），不用随机模糊。

**唯一例外**：被测契约本身就是定时/时序行为（例如流式 scanner 的 flush 间隔），此时 sleep 是断言的一部分而非「等业务发生」的手段。这类用例应显式在注释里注明「测试的正是这段定时行为」，或改用可控 clock。

## 约束

- `Test*` 函数禁 `import math/rand` 或第三方 fuzz 库做输入生成（`Benchmark*` 与 `crypto/rand` 生成测试证书不受限）。
- `Test*` 函数禁调用 `time.Sleep` 作为「等业务发生」的手段；等异步事件用 channel + `time.After` 超时兜底。
- 断言必须绑定具体期望值或状态；禁只断言「长度 > 0 / `err == nil` / 打了日志」。
- 循环次数需与被测契约挂钩（例如 N 个并发请求 → 断言 N 次结算）；禁用 `100_000` 次循环制造「压测感」。
- 例外：被测契约本身是定时行为时可用 `time.Sleep`，但必须在测试注释里说明这段 sleep 是断言的一部分。

## 例子

- 输入：一个纯函数 `CalcFee(usage)` 在多种 `usage` 组合下算出确定金额。
- 过程：禁写 `rng := rand.New(...); for i := 0; i < 10000; i++ { CalcFee(random(rng)) }`——通过率与正确性无关。改为几条 table case 手工列出 usage 与预期金额，用 `require.Equal` 断言。
- 预期结果：任意实现改动只要算法错就必然让某条 case 失败；随机压测转去 `BenchmarkCalcFee`。

## 验收

- [ ] `Test*` 内 grep `math/rand` 命中数为 0。
- [ ] `Test*` 内 grep `time\.Sleep` 命中数 ≤ 定时契约测试的数量，且每处都有注释说明。
- [ ] 断言均绑定具体期望值；grep `NotEmpty` / `Len.*>.*0` 用作值断言的处为 0。
- [ ] 循环规模与契约相关（N 并发 → N 断言）；无 `100000` 之类无契约大循环。
