# 可变形态 JSON 字段的首字节判形分派

## 目的

当一个 JSON 字段可能出现多种顶层形态时，用显式判形分派替代「先试 A 失败再试 B」的试错链，避免吞掉真实错误、避免每加一种形态都要在多处新增分支。

## 逻辑

对可变 shape 的字段，先以 `RawMessage` 收进内存，再调用「判形函数」查看修剪空白后的首字节，把它归类为 `object` | `array` | `string` | `boolean` | `null` | `number` 之一；随后按类型走对应分支的解码逻辑。

配套提供一个「RawMessage 转字符串」的解包函数：当原始 JSON 是字符串字面量时（首字符为 `"`）返回反引号解码后的字符串值；对 `null` 与空输入返回空串；对其他形态返回原始文本。

判形与解包这两个能力共同构成对「松散契约字段」的标准处理路径。禁止在同一字段上以「Unmarshal 是否成功」作为形态判据反复尝试解码。

## 约束

- 对可变 shape 字段的分派必须以「判形函数返回值 == 目标类型字符串」作为分支表达；禁止用 `Unmarshal` 的成功/失败作为形态判据。
- 判形函数对 `nil` 或全空白输入必须返回统一的「未知」占位字符串，禁 panic。
- 字符串解包函数：对首字符为 `"` 的输入必须返回反引号解码后的字符串值；对 `null` 与空输入返回空串；对其他形态返回原始文本。
- 调用点必须以判形函数的返回值作为分派开关；禁止调用点自己再实现一次基于 `len(raw)` / `raw[0]` 的判形。

## 例子

- 输入：上游返回的 `error` 字段既可能是 `{"code":..., "message":...}` 对象，也可能是 `"内部错误"` 字符串。
- 过程：以 `RawMessage` 接收该字段；调用判形函数拿到 `"object"` 或 `"string"`；据此进入对应分支——对象分支反序列化到 struct，字符串分支调用解包函数得到裸文本。
- 预期结果：两种上游形态都被正确归一化到内部错误模型；整个过程没有产生一次失败的反序列化调用，也没有 error 被静默吞掉。

## 验收

- [ ] 处理可变 shape 字段的分派点均以判形函数返回值做 `switch` / `if`，未出现试错式重复 `Unmarshal`。
- [ ] 判形函数对空输入返回统一的「未知」占位字符串（如 `"unknown"`），不 panic。
- [ ] 字符串解包函数对 `"foo"` 输入返回 `foo`；对 `null` 和空输入返回 `""`；对 `[1,2]` 或 `{"a":1}` 输入原样返回文本。
- [ ] 调用点未直接依赖 `raw[0]` 或 `len(raw)` 自行判形。
